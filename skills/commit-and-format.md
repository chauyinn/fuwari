# 中文排版与约定式提交规范 (Git Commit & Markdown Formatting Skill)

## 适用场景 (Applicability)
当 AI 助手（如 ChatGPT, Claude, Gemini, Cursor 等）在本项目中执行以下操作时，必须激活并严格遵循此技能：
1. **修改/撰写 Markdown 文档** (如 `.md`, `.mdx` 等文章和说明文件)。
2. **提交 Git 变更** (起草 commit 提交信息时)。

---

## 技能要求 (Skill Requirements)

### 1. 中文文案排版规范 (Chinese Copywriting Guidelines)
在修改或新增 Markdown 内容时，排版需遵循以下原则，保证中英文混排的美观度和专业性：

#### A. 空格使用规范 (Spacing)
*   **中英文之间加空格**：中文与英文单词、英文符号之间必须保留一个空格（半角空格）。
    *   *正确*：使用 Astro 框架和 Svelte 组件
    *   *错误*：使用Astro框架和Svelte组件
*   **数字与中文之间加空格**：中文与数字之间必须保留一个空格。
    *   *正确*：已发表 10 篇文章
    *   *错误*：已发表10篇文章
*   **数字与单位之间加空格**：数字与英文计量单位之间需保留空格。
    *   *正确*：宽带有 10 Gbps，磁盘空间 20 TB
    *   *错误*：宽带有 10Gbps，磁盘空间 20TB
    *   *例外*：百分比（%）和度数（°）与数字之间**不加**空格（如：`15%`、`90°`）。
*   **全角标点与相邻字符**：全角标点符号与其他字符（包括英文、数字、中文）之间**不留**空格。
*   **链接与加粗排版**：行内链接 `[文本](链接)`、加粗 `**文本**` 与相邻 of 中文汉字之间应保留一个空格。

#### B. 标点与全半角规范 (Punctuations, Full/Half Width)
*   **中文全角标点**：中文文本中必须使用全角标点符号（如 `，`、`。`、`！`、`？`、`：`、`；`、`（`、`）`）。
    *   *例外*：遇到完整的英文整句、特殊名词，其内容内部使用半角标点（如：乔布斯曾说「Stay hungry, stay foolish.」）。
*   **数字半角**：所有数字必须使用半角字符。
    *   *正确*：蛋糕只卖 1000 元
    *   *错误*：蛋糕只卖 １０００ 元
*   **直角引号**：简体中文推荐使用直角引号（`「` 和 `」`，内层使用 `『` 和 `』`）来代替弯引号（`“` 和 `”`）。
*   **不堆叠标点**：不要重复使用多个相同标点符号来表达情绪。
    *   *正确*：竟然发生了这种事！
    *   *错误*：竟然发生了这种事！！

#### C. 名词拼写规范 (Nouns & Capitalization)
*   **专有名词大小写**：必须使用正确的大小写拼写专有名词。
    *   *正确*：GitHub, macOS, Windows 11, Docker, Docker Compose, Astro, Git, YAML, JSON
    *   *错误*：github, Github, MacOS, docker, yaml, json
*   **不使用非正式缩写**：例如使用 `HTML5` 而不是 `H5`，`App` 而不是 `application`。

### 2. 中文约定式提交规范 (Conventional Commits in Chinese)
在进行 Git 提交时，提交信息（Commit Message）必须严格遵循约定式提交格式，且主体描述使用**中文**。

#### 提交信息格式：
```
<type>(<scope>): <subject>
```

*   **type (类型)**：必须为以下之一：
    *   `feat`：新功能、新组件、或添加新文章
    *   `fix`：修复 Bug 或处理错误（如代码报错、样式崩溃）
    *   `docs`：纯文档、文章内容的修改或补充（未涉及排版规范调整）
    *   `style`：排版或样式层面的调整（如按中文排版规范调整 Markdown 空格和标点，或格式化代码）
    *   `refactor`：代码重构（既不修复 Bug 也不添加新功能的代码更改）
    *   `chore`：构建过程、依赖项更新或辅助工具的变动（如更新 pnpm 依赖、修改配置文件、更新 `.gitmodules`）
*   **scope (范围)**：可选，说明本次修改的主要模块或文章名称（如 `post`, `theme`, `config`, `deps`）。
*   **subject (描述)**：必须使用**中文**进行简明扼要的说明，结尾**不要**加句号。

#### 示例 (Examples)：
*   `docs(post): 补充 Windows 远程桌面配置的文章内容`
*   `style(post): 按照排版规范格式化 macOS 配置 IPv6 文章`
*   `feat(post): 添加关于 Caddy 反代 RDP 的新文章`
*   `chore(deps): 更新 package.json 中的 Astro 依赖到最新版`
